# design-patterns-desafio
Repositório criado para exercitar alguns dos Padrões de Projeto existentes.
<br> O repositório de referência criado pelo Prof. Venilton encontra-se [aqui](https://github.com/digitalinnovationone/lab-padroes-projeto-java).
<br> A prática foi feita usando Java, sem fameworks ou outras ferramentas extras, utilizando conceitos do GOF (Gang of Four), 
com o intuito de entrar em contato com as características e particularidades de 3 padrões: 
- Singleton; 
- Strategy e 
- Facade.

---

## O que são Padrões de Projeto?
São soluções identificadas para problemas comuns no mundo de Desenvolvimento de Software. 
São como modelos para se guiar a fim de evitar ou resolver problemas de forma ágil.
<br> Os padrões funcionam como uma base, abstraíndo algumas cracterísticas gerais de soluções e implementações.

### Categorizados em (de acordo com seu próposito):
<b> Criacionais </b> - focados na forma de criação dos objetos para sua flexibilização e reutilização;
<br><b> Comportamentais </b> - tratam da comunicação e responsabilidades entre objetos ou classes;
<br><b> Estruturais </b> - preocupam-se em relacionar e combinar objetos para a criação de estruturas maiores que ainda sejam flexíveis.

## Estudados nesse repositório:

Singleton (Padrão Criacional)
	<br> Cria-se uma única instância de uma classe.
	Utilizado para evitar instanciar repetidamente um objeto.

Strategy (Padrão Comportamental)
	<br> Simplificar a variação de algoritmos para a resolução de um mesmo problema.

Facade (Padrão Estrutural)
	<br> Cria uma interface que simplifica a integração entre sistemas.
